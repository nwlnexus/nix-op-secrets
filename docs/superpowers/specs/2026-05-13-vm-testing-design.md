# VM-Based Integration Testing Design

## Goal

Add automated VM-based integration tests to `nix-op-secrets` that exercise the full activation path — real `op` CLI, real 1Password items, real file writes — inside an ephemeral Parallels Linux (NixOS) VM. Supplement with a documented manual guide for macOS VM testing (nix-darwin module path).

## Architecture

Two test paths:

| Path | Automation | Module tested |
|------|-----------|---------------|
| Linux (NixOS) | Fully automated script | `hmModules.default` via HM NixOS module |
| macOS | Manual guide + snapshot workflow | `darwinModules.default` via nix-darwin |

The Linux path creates a reproducible base VM from a NixOS ISO (no stored binary), clones it per test run, mounts the repo as a Parallels shared folder, runs `nixos-rebuild switch` with a stub flake, asserts file output, and tears down. All state is local to the repo directory (gitignored).

## Environment Setup

### `.env.sample`

```bash
# Copy this file to .env — NEVER commit .env
#
# 1. Generate a 1Password service account token:
#    https://developer.1password.com/docs/service-accounts/
#    The account needs read+write access to the test vault.
#
# 2. The test vault name is fixed at "nix-op-secrets-test".
#    Create it if it doesn't exist: op vault create nix-op-secrets-test
#
# 3. Place this file at the repo root as .env before running ./scripts/test-vm.sh

OP_SERVICE_ACCOUNT_TOKEN=ops_your_token_here
```

**Note:** The 1Password vault name is fixed at `nix-op-secrets-test` and is not configurable via `.env`. The `op://` URIs in `tests/vm/configuration.nix` and `tests/vm/fixtures/infra.env.tpl` hardcode this vault name. This keeps the stub flake reproducible without runtime templating.

## Prerequisites

Checked at the top of `test-vm.sh` with clear error messages:

| Prereq | Check |
|--------|-------|
| `prlctl` | `command -v prlctl` + `prlctl list` succeeds (confirms license) |
| `op` CLI | `command -v op` |
| `.env` | File exists at repo root |
| `OP_SERVICE_ACCOUNT_TOKEN` | Set and non-empty after sourcing `.env` |
| `op` auth | `op vault list` succeeds with the token (confirms connectivity) |
| `nix-op-secrets-test` vault exists | `op vault get nix-op-secrets-test` succeeds, or offer to create it |

## Base VM Setup (`scripts/vm/setup-base.sh`)

Idempotent — safe to run multiple times. Skips steps already completed. Checks `prlctl list` for `nix-op-secrets-base` before doing any work.

### SSH Key Generation

Generated before any VM work. Keys live in the repo (gitignored).

```bash
KEY_DIR="$(git rev-parse --show-toplevel)/tests/vm/keys"
mkdir -p "$KEY_DIR"
if [[ ! -f "$KEY_DIR/vm_key" ]]; then
  ssh-keygen -t ed25519 -C "nix-op-secrets-vm-test" -N "" -f "$KEY_DIR/vm_key"
fi
PUB_KEY="$(cat "$KEY_DIR/vm_key.pub")"
```

### NixOS Base VM Creation

1. **Download NixOS ISO** — NixOS 25.05 minimal, aarch64-linux, to `.cache/nixos.iso` (gitignored). Verified with SHA256 from the official NixOS release.

2. **Create VM**
   ```bash
   prlctl create "nix-op-secrets-base" \
     --ostype linux \
     --distribution nixos \
     --no-hdd
   prlctl set "nix-op-secrets-base" \
     --cpus 2 --memsize 4096 --device-add hdd --size 20480
   prlctl set "nix-op-secrets-base" \
     --device-set cdrom0 --image "$ISO_PATH" --connect
   prlctl start "nix-op-secrets-base"
   ```

3. **Unattended NixOS install via `prlctl exec`** — The NixOS live ISO runs as root with no password. Rather than relying on SSH into the live environment (which requires key pre-injection that the live ISO doesn't support), `prlctl exec` is used to run commands in the VM guest agent directly:

   ```bash
   # Wait for guest agent to come up
   until prlctl exec "nix-op-secrets-base" -- echo ready 2>/dev/null; do
     sleep 5
   done

   # Partition and format
   prlctl exec "nix-op-secrets-base" -- bash -c "
     parted /dev/sda -- mklabel gpt
     parted /dev/sda -- mkpart primary 512MB 100%
     parted /dev/sda -- mkpart ESP fat32 1MB 512MB
     parted /dev/sda -- set 2 esp on
     mkfs.ext4 -L nixos /dev/sda1
     mkfs.fat -F 32 -n boot /dev/sda2
     mount /dev/disk/by-label/nixos /mnt
     mkdir -p /mnt/boot
     mount -o umask=077 /dev/disk/by-label/boot /mnt/boot
   "

   # Write base config (with the generated public key baked in)
   CONFIG=$(sed "s|__VM_KEY_PUB__|$PUB_KEY|g" \
     "$REPO_ROOT/tests/vm/nixos-base.nix")
   prlctl exec "nix-op-secrets-base" -- bash -c "
     nixos-generate-config --root /mnt
     cat > /mnt/etc/nixos/configuration.nix <<'NIXEOF'
   $CONFIG
   NIXEOF
     nixos-install --no-root-passwd
     reboot
   "
   ```

   `tests/vm/nixos-base.nix` contains a `__VM_KEY_PUB__` placeholder that `setup-base.sh` substitutes with the actual public key before writing.

4. **`tests/vm/nixos-base.nix`** — committed NixOS config template for the base VM:
   - SSH enabled, port 22
   - `users.users.nixtest.openssh.authorizedKeys.keys = [ "__VM_KEY_PUB__" ]` (placeholder substituted by script)
   - `nix.settings.experimental-features = ["nix-command" "flakes"]`
   - `environment.systemPackages = [pkgs.git pkgs.jq]`
   - No passwords — SSH key only
   - `hardware.parallels.enable = true` — required for Parallels Tools (shared folders + guest agent)
   - `nixpkgs.config.allowUnfree = true` — required by `hardware.parallels`
   - `home-manager` not included at base VM level — it is wired via the test flake's NixOS module

5. **Wait for SSH after reboot** — Poll `ssh -i tests/vm/keys/vm_key -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null nixtest@<ip>` until connected (60s timeout with progress dots).

6. **Snapshot base state**
   ```bash
   prlctl stop "nix-op-secrets-base"
   prlctl snapshot create "nix-op-secrets-base" --name "clean"
   ```

## Test Run (`scripts/vm/run-tests.sh`)

**`trap cleanup EXIT` is the very first statement** — registered before any `op item create` call so that partial item creation is always cleaned up on failure.

```bash
cleanup() {
  prlctl stop "$TEST_VM" --kill 2>/dev/null || true
  prlctl delete "$TEST_VM" 2>/dev/null || true
  for ID in ${FIELD_ID:-} ${SSH_ID:-} ${DOC_ID:-}; do
    [[ -n "$ID" ]] && op item delete "$ID" --vault nix-op-secrets-test 2>/dev/null || true
  done
}
trap cleanup EXIT
```

**All SSH calls** use `-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null` to handle the rotated host key on each clone. This is safe because these are ephemeral local VMs on a trusted network.

### 1. Clone Base VM

```bash
TEST_VM="nix-op-secrets-test-$(date +%s)"
prlctl clone "nix-op-secrets-base" --name "$TEST_VM" --snapshot "clean"
prlctl start "$TEST_VM"
```

### 2. Mount Repo + Get IP

Parallels shared folders require Parallels Tools (`hardware.parallels.enable = true` in `nixos-base.nix`). The repo is mounted read-write so `nixos-rebuild` can write the lock file if needed.

```bash
REPO_ROOT="$(git rev-parse --show-toplevel)"
prlctl set "$TEST_VM" \
  --shf-host-add nix-op-secrets \
  --shf-host-path "$REPO_ROOT" \
  --shf-host-ro off

# prlctl exec requires guest agent (provided by hardware.parallels)
VM_IP=$(prlctl exec "$TEST_VM" -- hostname -I | awk '{print $1}')

# Poll SSH until ready
SSH_OPTS="-i $REPO_ROOT/tests/vm/keys/vm_key -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null"
until ssh $SSH_OPTS nixtest@"$VM_IP" true 2>/dev/null; do
  printf '.'; sleep 3
done
```

Repo available at `/media/psf/nix-op-secrets` inside the VM (Parallels Linux guest mount point).

### 3. Create Test 1Password Items

Items created with the host's `op` CLI using the service account token. IDs are stored in variables referenced by the `cleanup` trap.

```bash
FIELD_ID=$(op item create \
  --vault nix-op-secrets-test \
  --category Login \
  --title "nix-op-secrets-test-field" \
  "password=test-field-value" \
  --format json | jq -r '.id')

SSH_ID=$(op item create \
  --vault nix-op-secrets-test \
  --category "SSH Key" \
  --title "nix-op-secrets-test-ssh" \
  --format json | jq -r '.id')

DOC_ID=$(op document create "$REPO_ROOT/tests/vm/fixtures/test-doc.txt" \
  --vault nix-op-secrets-test \
  --title "nix-op-secrets-test-doc" \
  --format json | jq -r '.id')

# Template uses tests/vm/fixtures/infra.env.tpl — committed fixture
# Content: FIELD_VAL={{ op://nix-op-secrets-test/nix-op-secrets-test-field/password }}
# No item to create — op inject reads the field item directly
```

### 4. Inject Token into VM

The token is written to a file in the VM. `configuration.nix` sets `op-secrets.serviceAccountTokenFile = "/etc/op-secrets-test-token"`.

```bash
ssh $SSH_OPTS nixtest@"$VM_IP" \
  "printf '%s' '$OP_SERVICE_ACCOUNT_TOKEN' | sudo tee /etc/op-secrets-test-token > /dev/null"
```

### 5. Build and Switch

```bash
ssh $SSH_OPTS nixtest@"$VM_IP" \
  "sudo nixos-rebuild switch \
     --flake /media/psf/nix-op-secrets/tests/vm#test-vm 2>&1"
```

The stub flake's `configuration.nix` declares all four secret types (vault name is always `nix-op-secrets-test`):
- `test-field`: `op://nix-op-secrets-test/nix-op-secrets-test-field/password` → `/home/nixtest/.local/secrets/field.txt` (mode 0600)
- `test-ssh`: `op://nix-op-secrets-test/nix-op-secrets-test-ssh` (sshKey, writePublicKey=true) → `/home/nixtest/.ssh/test-vm-key` (mode 0600, pub 0644)
- `test-doc`: document vault=`nix-op-secrets-test` item=`nix-op-secrets-test-doc` → `/home/nixtest/.local/secrets/doc.txt` (mode 0600)
- `test-template`: template=`./fixtures/infra.env.tpl` → `/home/nixtest/.local/secrets/infra.env` (mode 0600)

### 6. Assertions

```bash
SSH="ssh $SSH_OPTS nixtest@$VM_IP"

fail() { echo "FAIL: $1" >&2; exit 1; }
check_file()    { $SSH "test -f '$1'" || fail "missing: $1"; }
check_mode()    { MODE=$($SSH "stat -c '%a' '$1'"); [[ "$MODE" == "$2" ]] || fail "mode $MODE != $2 on $1"; }
check_content() { $SSH "grep -q '$2' '$1'" || fail "content mismatch in $1 (expected: $2)"; }

check_file    "/home/nixtest/.local/secrets/field.txt"
check_mode    "/home/nixtest/.local/secrets/field.txt"     "600"
check_file    "/home/nixtest/.ssh/test-vm-key"
check_mode    "/home/nixtest/.ssh/test-vm-key"             "600"
check_file    "/home/nixtest/.ssh/test-vm-key.pub"
check_mode    "/home/nixtest/.ssh/test-vm-key.pub"         "644"
check_file    "/home/nixtest/.local/secrets/doc.txt"
check_mode    "/home/nixtest/.local/secrets/doc.txt"       "600"
check_file    "/home/nixtest/.local/secrets/infra.env"
check_content "/home/nixtest/.local/secrets/infra.env"     "test-field-value"

# Manifest — all five entries (including __pub synthetic key)
for KEY in test-field test-ssh "test-ssh__pub" test-doc test-template; do
  $SSH "jq -e '.\"$KEY\"' ~/.local/state/op-secrets/manifest.json" \
    || fail "manifest missing $KEY"
done

echo "All VM integration tests passed"
```

### 7. Teardown

Handled by the `cleanup` trap registered at the start of the script. Always runs, even on failure.

## Stub Flake (`tests/vm/flake.nix`)

```nix
{
  description = "nix-op-secrets VM integration test flake";

  inputs = {
    nixpkgs.url    = "github:nixos/nixpkgs/nixos-25.05";
    hm.url         = "github:nix-community/home-manager/release-25.05";
    hm.inputs.nixpkgs.follows = "nixpkgs";
    # op-secrets loaded from the mounted repo path at runtime — not pinned to GitHub
    op-secrets.url = "path:/media/psf/nix-op-secrets";
    op-secrets.inputs.nixpkgs.follows = "nixpkgs";
    darwin.url     = "github:nix-darwin/nix-darwin/nix-darwin-25.05";
    darwin.inputs.nixpkgs.follows = "nixpkgs";
  };

  outputs = { self, nixpkgs, hm, op-secrets, darwin, ... }: {
    # Linux automated test target
    nixosConfigurations.test-vm = nixpkgs.lib.nixosSystem {
      system = "aarch64-linux";
      modules = [
        ./nixos-base.nix
        hm.nixosModules.home-manager
        {
          home-manager.users.nixtest = {
            imports = [ op-secrets.hmModules.default ];
            home.stateVersion = "25.05";
          };
        }
        ./configuration.nix
      ];
    };

    # macOS manual test target — used by docs/vm-testing-macos.md
    # TODO: Requires a darwinSystem call. See docs/vm-testing-macos.md for the
    # required configuration shape. The nix-darwin module is tested manually;
    # this output is a placeholder until the macOS guide is fully wired up.
    darwinConfigurations.test-macos = darwin.lib.darwinSystem {
      system = "aarch64-darwin";
      modules = [
        hm.darwinModules.home-manager
        op-secrets.darwinModules.default
        ./configuration-macos.nix   # to be created per docs/vm-testing-macos.md
      ];
    };
  };
}
```

**Note on `path:` flake input:** `op-secrets.url = "path:/media/psf/nix-op-secrets"` resolves to the mounted repo inside the VM. This path does not exist on the developer's host machine — the `flake.lock` should be generated inside the VM after the shared folder is mounted, or the implementer should handle locking carefully (e.g., commit a lock that uses the GitHub URL and override the URL at `nixos-rebuild` time with `--override-input`).

## macOS VM Guide (`docs/vm-testing-macos.md`)

A committed markdown guide covering every step a developer must follow manually:

1. **Prerequisites** — Parallels Desktop ≥26, macOS IPSW for the target version (downloadable via `softwareupdate --fetch-full-installer`), `op` CLI, `.env` with `OP_SERVICE_ACCOUNT_TOKEN`

2. **Create the VM** — Parallels UI: New → Install macOS from IPSW → name `nix-op-secrets-macos-base` → 4 CPU / 8GB RAM / 60GB disk

3. **First boot** — Complete setup assistant, create user `nixtest`, enable Remote Login in System Settings → General → Sharing (or `sudo systemsetup -setremotelogin on` in Terminal)

4. **Install tooling in the VM:**
   - Nix: `curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install`
   - nix-darwin: follow https://github.com/nix-darwin/nix-darwin
   - 1Password CLI: `brew install 1password-cli`

5. **Configure SSH from host:**
   ```bash
   # On the host, after tests/vm/keys/ have been generated:
   ssh-copy-id -i tests/vm/keys/vm_key.pub nixtest@<vm-ip>
   # Or manually: append vm_key.pub to VM's ~/.ssh/authorized_keys
   ```

6. **Snapshot the base state:**
   ```bash
   prlctl snapshot create nix-op-secrets-macos-base --name "clean"
   ```

7. **Create test 1Password items** (same as Linux step 3, run from host):
   ```bash
   source .env
   op item create --vault nix-op-secrets-test --category Login \
     --title "nix-op-secrets-test-field" "password=test-field-value"
   # ... SSH Key, Document as in Linux guide
   ```

8. **Inject token into the VM:**
   ```bash
   SSH_OPTS="-i tests/vm/keys/vm_key -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null"
   ssh $SSH_OPTS nixtest@<vm-ip> \
     "printf '%s' '$OP_SERVICE_ACCOUNT_TOKEN' | sudo tee /etc/op-secrets-test-token > /dev/null"
   ```

9. **Mount repo and run darwin-rebuild:**
   ```bash
   prlctl set nix-op-secrets-macos-base \
     --shf-host-add nix-op-secrets \
     --shf-host-path "$(pwd)"
   # Repo available at /Volumes/nix-op-secrets inside macOS VM
   ssh $SSH_OPTS nixtest@<vm-ip> \
     "sudo darwin-rebuild switch --flake /Volumes/nix-op-secrets/tests/vm#test-macos"
   ```

10. **Assertions** (same checks as Linux, run over SSH)

11. **Cleanup:**
    ```bash
    prlctl snapshot revert nix-op-secrets-macos-base "clean"
    op item delete <field-id> --vault nix-op-secrets-test
    # ... delete other items
    ```

## Error Handling

- All scripts use `set -euo pipefail`
- `trap cleanup EXIT` is registered as the **first statement** in `run-tests.sh`, before any `op item create` — guarantees VM and 1Password item cleanup even on partial failure
- Each prereq check prints a specific, actionable error message before exiting
- SSH connectivity polling has a configurable timeout (default 60s) with progress dots
- VM creation uses `prlctl exec` (guest agent) rather than live ISO SSH — avoids the key-injection bootstrapping problem
- All SSH calls use `-o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null` — ephemeral VMs rotate host keys per clone; this is safe for local trusted VMs
