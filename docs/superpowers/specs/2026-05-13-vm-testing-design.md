# VM-Based Integration Testing Design

## Goal

Add automated VM-based integration tests to `nix-op-secrets` that exercise the full activation path — real `op` CLI, real 1Password items, real file writes — inside an ephemeral Parallels Linux (NixOS) VM. Supplement with a documented manual guide for macOS VM testing (nix-darwin module path).

## Architecture

Two test paths:

| Path | Automation | Module tested |
|------|-----------|---------------|
| Linux (NixOS) | Fully automated script | `hmModules.default` via HM NixOS module |
| macOS | Manual guide + snapshot workflow | `darwinModules.default` via nix-darwin |

The Linux path creates a reproducible base VM from a NixOS ISO (no stored binary), clones it per test run, mounts the repo as a Parallels shared folder, runs `nixos-rebuild switch` with a stub flake, asserts file output, and tears down. All state is local to the repo directory (gitignored).

## Environment Setup

### `.env.sample`

```bash
# Copy this file to .env — NEVER commit .env
#

#    https://developer.1password.com/docs/service-accounts/

OP_TEST_VAULT=nix-op-secrets-test
```

## Prerequisites

Checked at the top of `test-vm.sh` with clear error messages:

| Prereq | Check |
|--------|-------|
| `prlctl` | `command -v prlctl` + `prlctl list` succeeds (confirms license) |
| `op` CLI | `command -v op` |
| `.env` | File exists at repo root |
| `OP_SERVICE_ACCOUNT_TOKEN` | Set and non-empty after sourcing `.env` |
| `op` auth | `op vault list` succeeds with the token (confirms connectivity) |
| `OP_TEST_VAULT` exists | `op vault get "$OP_TEST_VAULT"` succeeds, or offer to create it |

## Base VM Setup (`scripts/vm/setup-base.sh`)

Idempotent — safe to run multiple times. Skips steps already completed.

### SSH Key Generation

```bash
KEY_DIR="$(git rev-parse --show-toplevel)/tests/vm/keys"
mkdir -p "$KEY_DIR"
if [[ ! -f "$KEY_DIR/vm_key" ]]; then
  ssh-keygen -t ed25519 -C "nix-op-secrets-vm-test" -N "" -f "$KEY_DIR/vm_key"
fi
```

### NixOS Base VM Creation

1. **Download NixOS ISO** — NixOS 25.05 minimal, aarch64-linux, to `"$(git rev-parse --show-toplevel)/.cache/nixos.iso"` (gitignored). Verified with SHA256.

2. **Create VM**
   ```bash
   prlctl create "nix-op-secrets-base" \
     --ostype linux \
     --distribution nixos \
     --no-hdd
   prlctl set "nix-op-secrets-base" \
     --cpus 2 --memsize 4096 --device-add hdd --size 20480
   prlctl set "nix-op-secrets-base" \
     --device-set cdrom0 --image "$ISO_PATH" --connect
   ```

3. **Boot and unattended NixOS install** — The script SSHs into the NixOS live environment (default SSH is enabled on the live ISO) and runs:
   - Partition + format disk
   - Write `tests/vm/nixos-base.nix` as `/mnt/etc/nixos/configuration.nix`
   - `nixos-install --no-root-passwd`
   - Reboot into installed system

4. **`tests/vm/nixos-base.nix`** — committed NixOS config for the base VM:
   - SSH enabled, port 22
   - `vm_key.pub` injected into `users.users.nixtest.openssh.authorizedKeys.keys`
   - `nix.settings.experimental-features = ["nix-command" "flakes"]`
   - `environment.systemPackages = [pkgs.git pkgs.jq]`
   - No passwords — SSH key only
   - `home-manager` as a NixOS module (via flake input)

5. **Wait for SSH** — Poll `ssh -i tests/vm/keys/vm_key` until connected (60s timeout)

6. **Snapshot base state**
   ```bash
   prlctl stop "nix-op-secrets-base"
   prlctl snapshot create "nix-op-secrets-base" --name "clean"
   ```

## Test Run (`scripts/vm/run-tests.sh`)

### 1. Clone Base VM

```bash
TEST_VM="nix-op-secrets-test-$(date +%s)"
prlctl clone "nix-op-secrets-base" --name "$TEST_VM" --snapshot "clean"
prlctl start "$TEST_VM"
```

### 2. Mount Repo + Get IP

```bash
REPO_ROOT="$(git rev-parse --show-toplevel)"
prlctl set "$TEST_VM" \
  --shf-host-add nix-op-secrets \
  --shf-host-path "$REPO_ROOT" \
  --shf-host-ro off
VM_IP=$(prlctl exec "$TEST_VM" -- hostname -I | awk '{print $1}')
# Poll SSH until ready (30s timeout)
```

Repo available at `/media/psf/nix-op-secrets` inside the VM.

### 3. Create Test 1Password Items

All items prefixed `nix-op-secrets-test-` and created in `$OP_TEST_VAULT`. IDs captured for cleanup.

```bash
# Field secret
FIELD_ID=$(op item create \
  --vault "$OP_TEST_VAULT" \
  --category Login \
  --title "nix-op-secrets-test-field" \
  "password=test-field-value" \
  --format json | jq -r '.id')

# SSH Key secret (op creates with /private key and /public key fields)
SSH_ID=$(op item create \
  --vault "$OP_TEST_VAULT" \
  --category "SSH Key" \
  --title "nix-op-secrets-test-ssh" \
  --format json | jq -r '.id')

# Document secret
DOC_ID=$(op document create "$REPO_ROOT/tests/vm/fixtures/test-doc.txt" \
  --vault "$OP_TEST_VAULT" \
  --title "nix-op-secrets-test-doc" \
  --format json | jq -r '.id')

# Template secret — uses a field from the field item above
# (template file is at tests/vm/fixtures/infra.env.tpl, committed)
```

### 4. Inject Token into VM

```bash
ssh -i tests/vm/keys/vm_key nixtest@"$VM_IP" \
  "echo 'OP_SERVICE_ACCOUNT_TOKEN=$OP_SERVICE_ACCOUNT_TOKEN' | \
   sudo tee /etc/op-secrets-test-token > /dev/null"
```

The stub `configuration.nix` sets `op-secrets.serviceAccountTokenFile = "/etc/op-secrets-test-token"`.

### 5. Build and Switch

```bash
ssh -i tests/vm/keys/vm_key nixtest@"$VM_IP" \
  "sudo nixos-rebuild switch \
     --flake /media/psf/nix-op-secrets/tests/vm#test-vm 2>&1"
```

The stub flake's `configuration.nix` declares all four secret types:
- `test-field`: `op://nix-op-secrets-test/nix-op-secrets-test-field/password` → `/home/nixtest/.local/secrets/field.txt`
- `test-ssh`: `op://nix-op-secrets-test/nix-op-secrets-test-ssh` (sshKey, writePublicKey=true) → `/home/nixtest/.ssh/test-vm-key`
- `test-doc`: document from `$OP_TEST_VAULT` / `nix-op-secrets-test-doc` → `/home/nixtest/.local/secrets/doc.txt`
- `test-template`: template → `/home/nixtest/.local/secrets/infra.env`

### 6. Assertions

```bash
SSH="ssh -i tests/vm/keys/vm_key nixtest@$VM_IP"

check_file()  { $SSH "test -f '$1'" || fail "missing: $1"; }
check_mode()  { MODE=$($SSH "stat -c '%a' '$1'"); [[ "$MODE" == "$2" ]] || fail "mode $MODE != $2 on $1"; }
check_content() { $SSH "grep -q '$2' '$1'" || fail "content check failed: $1"; }

check_file  "/home/nixtest/.local/secrets/field.txt"
check_mode  "/home/nixtest/.local/secrets/field.txt"       "600"
check_file  "/home/nixtest/.ssh/test-vm-key"
check_mode  "/home/nixtest/.ssh/test-vm-key"               "600"
check_file  "/home/nixtest/.ssh/test-vm-key.pub"
check_mode  "/home/nixtest/.ssh/test-vm-key.pub"           "644"
check_file  "/home/nixtest/.local/secrets/doc.txt"
check_mode  "/home/nixtest/.local/secrets/doc.txt"         "600"
check_file  "/home/nixtest/.local/secrets/infra.env"
check_content "/home/nixtest/.local/secrets/infra.env"     "test-field-value"

# Manifest
$SSH "jq -e '.\"test-field\"' ~/.local/state/op-secrets/manifest.json" || fail "manifest missing test-field"
$SSH "jq -e '.\"test-ssh\"' ~/.local/state/op-secrets/manifest.json"   || fail "manifest missing test-ssh"
$SSH "jq -e '.\"test-ssh__pub\"' ~/.local/state/op-secrets/manifest.json" || fail "manifest missing test-ssh__pub"
$SSH "jq -e '.\"test-doc\"' ~/.local/state/op-secrets/manifest.json"   || fail "manifest missing test-doc"
$SSH "jq -e '.\"test-template\"' ~/.local/state/op-secrets/manifest.json" || fail "manifest missing test-template"
```

### 7. Teardown (trap EXIT — always runs)

```bash
prlctl stop "$TEST_VM" --kill 2>/dev/null || true
prlctl delete "$TEST_VM" 2>/dev/null || true

for ID in $FIELD_ID $SSH_ID $DOC_ID; do
  op item delete "$ID" --vault "$OP_TEST_VAULT" 2>/dev/null || true
done
```

## Stub Flake (`tests/vm/flake.nix`)

```nix
{
  description = "nix-op-secrets VM integration test flake";

  inputs = {
    nixpkgs.url     = "github:nixos/nixpkgs/nixos-25.05";
    hm.url          = "github:nix-community/home-manager/release-25.05";
    hm.inputs.nixpkgs.follows = "nixpkgs";
    # op-secrets loaded from the mounted repo path — not from GitHub
    op-secrets.url  = "path:/media/psf/nix-op-secrets";
    op-secrets.inputs.nixpkgs.follows = "nixpkgs";
  };

  outputs = { self, nixpkgs, hm, op-secrets, ... }: {
    nixosConfigurations.test-vm = nixpkgs.lib.nixosSystem {
      system = "aarch64-linux";
      modules = [
        ./nixos-base.nix
        hm.nixosModules.home-manager
        {
          home-manager.users.nixtest = {
            imports = [ op-secrets.hmModules.default ];
            home.stateVersion = "25.05";
            # op-secrets config defined in configuration.nix
          };
        }
        ./configuration.nix
      ];
    };

    # macOS output — used by the manual macOS guide
    darwinConfigurations.test-macos = /* ... */;
  };
}
```

## macOS VM Guide (`docs/vm-testing-macos.md`)

A committed markdown guide covering:

1. **Prerequisites** — Parallels Desktop, macOS IPSW from Apple, `op` CLI, `.env` with token
2. **Create the VM** — Parallels UI steps: New → Install macOS from IPSW → name `nix-op-secrets-macos-base` → 4 CPU / 8GB / 60GB
3. **First boot** — Complete setup assistant, create user `nixtest`, enable SSH (`sudo systemsetup -setremotelogin on`)
4. **Install tooling** — Determinate Systems Nix, nix-darwin, `brew install 1password-cli`
5. **Configure SSH** — Copy `tests/vm/keys/vm_key.pub` to VM's `~/.ssh/authorized_keys`
6. **Snapshot** — `prlctl snapshot create nix-op-secrets-macos-base --name "clean"`
7. **Running the test** — Step-by-step commands: revert snapshot → start → mount repo → SSH in → `darwin-rebuild switch --flake /Volumes/nix-op-secrets/tests/vm#test-macos` → verify assertions → revert
8. **Assertions** — Same file/permission/manifest checks listed in table form
9. **Cleanup** — Revert snapshot, delete test 1Password items via `op item delete`

## Error Handling

- All scripts use `set -euo pipefail`
- `trap` on EXIT for teardown (VM destroy + 1Password item cleanup always runs)
- Each prereq check prints a specific, actionable error message
- SSH connectivity polling has a configurable timeout (default 60s) with progress dots
- Partial test failures print which assertion failed before teardown
